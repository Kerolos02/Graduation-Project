TruckMate Backend API Endpoints
NOTE: For full request/response schemas, Flutter mapping hints, and Endpoint IDs,
      see FLUTTER_API_REFERENCE.txt in this same folder.

Base URL
- Production: http://truckmateapi.runasp.net

Authentication
- Use JWT Bearer token in header for protected APIs:
  Authorization: Bearer {token}
- Logout endpoint (Driver + Trader):
  POST /api/auth/logout   (Authorized)

Health / Infra
- GET /                      (Public)  -> API root check
- GET /health                (Public)  -> health check
- SignalR Hub: /hubs/driver  (JWT required, token can be sent as access_token query)

Auth
- POST /register/send-otp          (Public)
- POST /register/verify-otp        (Public)  -> body: { email, otp } or { email, otpCode, otpToken? }; returns verificationToken
- POST /register                   (Public)  -> accepts verificationToken OR otpToken+otpCode OR email+otp in registration body
- POST /register/forgot-password   (Public)
- POST /register/verify-reset-otp  (Public)
- POST /register/reset-password    (Public)
- POST /login                      (Public)

Driver Marketplace Trips (browse & accept open requests — separate from 45s offer flow)
- GET  /api/driver/trips/available-requests?page=1&pageSize=20&sortBy=posted_desc|price_desc|distance_asc  (Driver role, online only; empty if driver has active trip)
- GET  /api/driver/trips/requests/{requestId}      (Driver role, online only; detail includes trader phone + special notes)
- POST /api/driver/trips/requests/{requestId}/accept   (Driver role; 409 if active trip / race; 410 if expired/cancelled)
- POST /api/driver/trips/requests/{requestId}/reject   (Driver role; body JSON optional { "reason": "..." })
- GET  /api/driver/trips/my-trips?status=all|active|completed&page=1&pageSize=10  (Driver role)

Driver Wallet

- GET   /api/driver/wallet/trips?filter=all|this_week|this_month&page=1&pageSize=10     (Driver role)

SignalR Server -> Client events (DriverHub)
- NewTripOffer
- TripOfferExpired
- TripAccepted
- TripOfferCancelled
- StatusAcknowledged
- TripAssigned
- DriverAvailabilityChanged
- CourierTripStarted
- DriverAcceptedTrip
- DriverDeclinedTrip
- DriverAssigned
- NoDriversAvailable
- NewRequestAvailable          (marketplace: new open TripRequest in driver zone)
- RequestTaken                 (marketplace: { requestId, requestNumber } — remove card for other drivers)
- RequestCancelled             (marketplace: { requestId, reason })
- RequestExpired               (marketplace: { requestId } — all drivers in marketplace-all-drivers group)
- DriverAccepted               (trader group: { driverName, acceptedAt } — same hub, trader clients)
- RequestAccepted              (dispatcher group: { requestId, driverId })
- RequestExpiredNoDriver       (dispatcher group: { requestId })
- DriverRejectedRequest        (dispatcher group: { requestId, driverId })
